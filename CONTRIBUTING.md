# Contribuir

1. **Fork y rama**: cree una rama descriptiva desde `main`.
2. **Estilo**: mantenga PHP 8.x estándar y documente bloques complejos con comentarios breves.
3. **Logs y backups**: no suba archivos del directorio `my_ia_site/logs/` ni copias `.bkp`. Utilice `bkp/` para respaldos locales.
4. **Pruebas**: describa en el PR cómo verificó los cambios. Para módulos web, indique rutas visitadas o scripts ejecutados.
5. **Revisión**: envíe un PR y espere la revisión de otro colaborador antes de fusionar.

Gracias por colaborar.
